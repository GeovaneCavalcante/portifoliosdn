Controladores SDN


No paradigma de Redes Definidas por Software o Controlador tem uma função de extrema importância, ele tem o papel de analisar fluxos enviados pelos equipamentos conectados a ele. Todo pacote que é recebido por esses equipamentos e o mesmo não saibam para onde encaminhá-lo é enviado para Controlador. O mesmo analisa o pacote recebido e define e implanta uma nova regra no plano de dados, caso seja necessários. Na próxima requisição mandada aos equipamentos, será verificado novamente o cabeçalho, se o fluxo já atender uma regra existe será usada, caso contrário, novamente será enviado para o controlador analisá-lo e definir seu destino (MORAIS, 2015, p. 23), conforme a figura 4.

Figura 2


Os Controladores SDN utilizam diversas linguagens de programação como C, C++, Python, Java e etc. O primeiro Controlador foi chamado NOX,  um controlador que oferece uma API que permite o desenvolvimento de ferramentas para alterar ou monitorar o comportamento da rede OpenFlow (MARQUES, 2013, p. 11). Implementado com C++ e a partir dele vários outros controladores foram construídos, cada um deles com suas características de uso, por isso a escolha do controlador para redes deve ser de extrema atenção, levando em consideração as características de cada um. A implementação dos Controladores, pode ser feita em um servidor na rede, onde o mesmo será conectado a todos equipamentos necessários. (MORAIS, 2015, p. 24).
